Authentication Bypass: Incomplete Webhook Verification
Active — detected May 20, 2026. Present in 2 consecutive scans. No code changes have addressed this finding.
RepoDryRunSec/dryrun-api
PR#412 — fix/auth-bypass-recovery
AgentGeneral Security Analyzer
Filepkg/usecases/providers/webhook_verifier.go

Description

The webhook verification handler checks the signature only when a X-Hub-Signature-256 header is present. Requests without the header bypass verification entirely, allowing unauthenticated webhook delivery to trigger internal processing logic.

Evidence

In VerifyWebhook(), the conditional if sig := r.Header.Get(...) returns early only on mismatch — a missing header evaluates the check as falsy and proceeds without validation.

Recommendation

Require the signature header unconditionally. Return a 401 if the header is absent rather than treating absence as a passing state.
